# Product roadmap

High-level phases. Detailed rules live in [BUSINESS_RULES.md](./BUSINESS_RULES.md) and [CAMPAIGNS.md](./CAMPAIGNS.md).

**North star today:** first venue → first customer → first redemption → first paying customer.

## Shipped

| Phase | What |
| --- | --- |
| **1 — Loyalty foundation** | QR join, wallet, milestones, multi-venue workspace, owner dashboard, venue listing approval |
| **1b — NFC + self-redeem (2026 pivot)** | NFC stamp stands, slide-to-redeem; removed staff scanner, claim QR, team invites, stamp QR |
| **1.5 — Retention basics** | Last visit, visit count, rewards claimed, joined date on customer list |
| **2 — Profiles** | Customer profile, visit/reward history, notes, birthday |
| **3 — Timeline** | Unified activity timeline on profile |
| **4 — Campaigns** | Happy Hour, Quiet Day, VIP, Bring Back — stamp multipliers on owner `/campaigns` |

## In progress

| Phase | What |
| --- | --- |
| **Pilot validation** | One real venue live for 2+ weeks; measure repeat visits and redemptions |
| **1 polish** | Email deliverability, NFC edge cases, onboarding copy |

## Planned (next)

| Phase | What |
| --- | --- |
| **5 — Communication** | Push notifications in mobile app (inbox UI exists; delivery not wired) |
| **Birthday campaign** | Template for birthday-week stamp boost |

## Future (after proven retention)

Billing per venue, deeper analytics, multi-location dashboards, POS integrations, segmentation, marketing automation.

Build order: daily usage → retention → communication → analytics → operations. No large feature sets before demand is proven.
