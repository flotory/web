# Flotory v2 — Universal customer QR

**Status:** Phases 1–3 **shipped** in code. Phase 4 (auto-join policy) is product-default **silent join on first scan**.

## What changed

| Before | After |
|--------|--------|
| One `customers.qr_token` per venue card | One `user_stamp_tokens.public_token` per user |
| QR on each wallet card | **My QR** tab (mobile) / `/my-qr` (web) |
| Scanner matched card UUID at venue | Scanner resolves **user → card at scanner venue** (auto-join) |
| Claim QR | Unchanged (`flotory:redeem:{token}`) |

## Stamp QR payload

```text
flotory:member:{public_token}
```

Parsed by `App\Support\LoyaltyQr` and `resources/js/lib/loyaltyQr.ts`.

## Configuration

| Env | Default | Meaning |
|-----|---------|---------|
| `LOYALTY_UNIVERSAL_QR` | `true` | Use `user_stamp_tokens` for stamp scans |
| `LOYALTY_LEGACY_CARD_QR` | `false` | Accept old per-card `customers.qr_token` at scanner |
| `LOYALTY_LEGACY_CARD_QR_SUNSET_AT` | *(empty)* | Optional ISO datetime; after this, legacy is off even if enabled |

## Migration commands

```bash
php artisan migrate
php artisan app:backfill-user-stamp-tokens
```

## Staff scanner

- **Camera:** customer **My QR** or claim QR.
- **Manual fallback:** `POST .../scanner/stamps` with `customer_id` (no card QR needed).
- **Legacy card QR:** only if `LOYALTY_LEGACY_CARD_QR=true`.

## API

| Endpoint | Role |
|----------|------|
| `GET /api/customer/stamp-qr` | Customer — universal stamp QR |
| `POST /api/venues/{venue}/scanner/scan` | Staff — stamp or redeem |
| Customer card payloads | **`qr_token` omitted** (hidden on `Customer` model) |

## Rollback

Set `LOYALTY_LEGACY_CARD_QR=true` temporarily. Customer apps already point guests to My QR.
